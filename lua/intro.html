The Bolt Programming Language

Lua is an amazing language, but it isn't perfect. Being a compiled language but not being able to run standalone executables is a strange concept to make with. Bolt is designed to be exactly like Lua with language optimizations that make it closer to TypeScript. Here are the main differences:

  • Lua is a compiled language with no standalone executables.
    Bolt is a transpiled language that transpiles to C++, then uses the gcc compiler to make an executable.
  • Lua does not deal with types outside of type().
    Bolt allows for explicit types, but does not force them. If a type is mismatched, it delivers a warning at compile time.
  • Lua releases versions of their source every couple years.
    Bolt puts out a new release every month that can optionally be upgraded automatically with bolt --upgrade.

    • Note that Bolt is not ready for stable use. It's currently reported version is BTC version 0.0.2, build 392 (testing).
